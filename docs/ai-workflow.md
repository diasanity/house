# 代銷 AI 工作台｜系統化工作流程 v1

這份文件把日常代銷工作拆成可被 AI 呼叫的固定流程，目標是把「每次重新做」改成「呼叫固定模組」。

## 一、工作台總目標

建立一套可在案場、內部會議、土地開發、價格策略與行銷輸出中反覆使用的 AI 工作台。

使用者只要輸入業主、地號、案名、底價表、客戶抗性或會議資料，系統就依固定流程輸出：

- 土地評估 JSON
- 一頁式土地評估表
- 競案分析
- 價格策略與 ABC 分級
- 行銷文案
- 客戶話術
- 專案簡報架構
- 待複核事項

---

## 二、五層 AI 工作流程

### Phase 1｜資訊輸入 Input Layer

使用者可直接貼入：

- 地號
- 業主名稱
- 土地售價
- PDF 文字
- 截圖轉錄文字
- 謄本資訊
- 地籍圖資訊
- 都市計畫資料
- 競案名稱
- 底價表資料
- 客戶背景
- 會議紀錄

AI 必須先將資料整理為：

1. 已知資料
2. 待查資料
3. 待複核資料
4. 可直接進入報告欄位的資料

---

### Phase 2｜分析引擎 Analysis Layer

核心模組：

1. 土地評估
2. 競案分析
3. 價格策略
4. 產品規劃
5. 客群判斷

此階段重點不是寫漂亮文字，而是把資料轉為可決策的結論。

---

### Phase 3｜策略決策 Strategy Layer

AI 需要輸出：

- 本案市場位置
- 價格上緣
- 順銷安全帶
- 首波成交帶
- 主力產品
- 不建議產品
- 對業主可說的策略理由
- 對主管可說的簡短結論

---

### Phase 4｜對外輸出 Execution Layer

把分析結果轉成可直接使用的銷售材料：

- FB 投廣文案
- Threads 自然文
- LINE 客戶通知
- 簡訊
- 業務話術
- 回訪邀約
- 簡報標題與逐頁內容

---

### Phase 5｜模板沉澱 System Layer

每一次完成的結果，都要能沉澱為模板：

- 土地評估模板
- 競案分析模板
- 底價分級模板
- 行銷文案模板
- 客戶話術模板
- 簡報模板

未來不再重新思考格式，只更新資料與判斷。

## 四、第一階段優先開發

### v1.0｜土地評估模組

已存在 App 基礎功能：

- JSON 貼上
- 匯入 JSON
- 欄位手動修改
- 報告預覽
- 列印 / PDF

下一步強化：

1. 增加「一鍵產生 ChatGPT 調研指令」
2. 增加「固定 JSON Schema」
3. 增加「待複核事項」欄位強化
4. 增加「競案表格可新增／刪除」

## 六、操作原則

1. 所有輸出都要能直接被使用。
2. 土地評估預設輸出 JSON。
3. 資料不足時，欄位內標註「待提供／待複核／依公開資料初判／依業主資料初判」。
4. 不能硬填無法確認的資料。
5. 需要市場資料時，應以最新公開資料與實價登錄邏輯為核心。
6. 結論要能直接轉成業主提報、主管簡報或案場執行。
