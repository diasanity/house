# 代銷 AI 工作台｜固定工作原則

本文件記錄使用者對代銷 AI 工作台的長期要求，後續所有功能規劃、Prompt、JSON Schema、UI 與報告輸出都需依此原則持續精進。

## 一、核心目標

把使用者在代銷工作中反覆詢問、反覆修正、反覆使用的判斷邏輯，逐步整理成可被工具化、模組化、流程化的 AI 工作台。

目標不是只做單次回答，而是把每次工作沉澱成：

- 固定流程
- 固定欄位
- 固定 Prompt
- 固定 JSON Schema
- 可手動修改的工具介面
- 可交付業主或主管的輸出格式

---

## 二、資訊收集原則

後續只要使用者在代銷工作中提出問題，都要判斷是否可沉澱進工具，例如：

- 土地評估
- 基地四向現況判讀
- 建蔽率、容積率、分區查找
- 學區與里別查核
- 競案篩選
- 實價登錄成交均價
- 車位價格判斷
- 價格策略
- 底價表分級
- 係數邏輯
- 行銷文案
- 投廣策略
- 客戶話術
- 交屋通知
- 簡報架構

若某次對話出現可重複使用的規則，應整理為後續模組的改進項目。

---

## 三、求證與資料品質原則

所有輸出必須反覆確認求證，尤其是以下資料：

1. 土地面積：需逐筆地號確認，不可用區域推估。
2. 建蔽率與容積率：需依都市計畫書、細部計畫、分區管制要點或政府公開資料查證。
3. 里別：需以地籍位置、里界圖、戶政或區公所資料交叉判斷。
4. 學區：需查教育局最新年度公告，國小與國中分開列明。
5. 基地現況：需依實際地圖、衛星圖、街景、地籍圖或使用者提供圖資逐向判讀。
6. 競案與行情：以實價登錄成交資料為核心，優先近半年，不足則近一年。
7. 車位價格：以周邊競案實際車位行情為核心，公式推算只做合理性檢查。

無法確認時，不得硬填，需在欄位中標註：

- 待提供
- 待複核
- 依公開資料初判
- 依業主資料初判

---

## 四、工具精進原則

每次新增想法時，優先判斷是否屬於以下其中一類：

### 1. Prompt 改進

例如讓 ChatGPT 更穩定輸出土地評估 JSON、價格策略、行銷文案或客戶話術。

### 2. JSON Schema 改進

例如增加欄位、調整資料層級、讓 App 更容易匯入。

### 3. UI 改進

例如手機版更好操作、欄位更清楚、競案可新增刪除、基地面積可逐筆編輯。

### 4. 報告輸出改進

例如一頁式土地評估表、PDF 檔名、Excel 版型、業主簡報格式。

### 5. 資料查核流程改進

例如加入資料來源、複核清單、查核優先順序。

---

## 五、開發優先順序

目前工具優先順序如下：

1. 土地評估模組
2. 價格策略模組
3. 競案分析模組
4. 行銷輸出模組
5. 客戶話術模組
6. 專案簡報模組
7. 模板庫

每一個模組都需做到：

- 可輸入
- 可編輯
- 可輸出
- 可複製 JSON
- 可沉澱模板

---

## 六、重要工作態度

使用者授權 AI 大膽提出工具優化想法，但所有涉及資料事實、價格行情、土地法規、市場判斷、競案數據與對外報告的內容，都必須查證、註明來源或標註待複核。

AI 可以主動提出優化方向，但不應憑空擴大承諾，也不應把未確認資訊包裝成確定結論。
